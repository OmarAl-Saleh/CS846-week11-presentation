# Problem C: Incident Reporter — Performance Optimization

## Task Description

You are given a fully functional Incident Reporter written in Python. The program generates a synthetic incident/event log and produces a report with multiple metrics (endpoint stats, device/browser mix, suspicious-client heuristics, etc.).

**The code is functionally correct — all test cases pass.** However, it has performance issues that make it slower than necessary on larger datasets.

**Your task:** Optimize the code to run as fast as possible **without breaking correctness**. All existing test cases must continue to pass after your changes.

## Instructions

1. To run the code:
   ```
   python incident_reporter.py
   ```
2. Observe that all correctness tests pass, and note the timing output for the benchmark.
3. Try to optimize the code and reduce the timing, whil ensuring all test cases pass.
